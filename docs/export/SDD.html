3. Přehled systému

Chronis je webová aplikace pro správu docházky a absencí. Skládá se z veřejné prezentační části a interní systémové části přístupné po přihlášení.

Frontend

  • React — UI framework (SPA)
  • React Router — klientský routing
  • Vite — vývojový server a build
  • Tailwind CSS — stylování
  • Lucide React — ikony

Backend

  • PHP — REST API endpointy
  • PDO — práce s databází
  • PHPMailer — e-maily přes SMTP
  • Session — autentizace

Infrastruktura

  • MySQL — relační databáze
  • WAMP — lokální PHP server
  • Node.js + npm — frontend tooling
  • Windows — vývojové prostředí

Backend návrh

API endpointy, autentizace, autorizace a klíčové pomocné soubory.

7.1 Připojení a autentizace

connection.php vytváří PDO připojení k MySQL databázi chronisdb (server localhost, uživatel root, lokální WAMP). Po přihlášení ukládá login.php do session: ID, jméno, příjmení, login, pozici, příznak administrátora a příznak vedoucího. Endpoint get_user.php vrací přihlášeného uživatele, logout.php session ruší.

Sekce 9–14

Procesy, bezpečnost a nasazení

Návrh klíčových procesů, bezpečnostní mechanismy, exporty a postup spuštění.

9. Návrh hlavních procesů

9.1 Přihlášení uživatele
1

Uživatel vyplní přihlašovací formulář

Frontend odešle POST na login.php s přihlašovacím jménem a heslem.

2

Backend ověří přihlašovací údaje

login.php vyhledá uživatele v DB a ověří heslo přes password_verify.

3

Session a přesměrování

Při úspěchu se uloží identita do session. Frontend přesměruje do /system_main.

9.2 Evidence docházky (stavový automat)
1

Příchod → stav: V práci

Zaznamenán čas příchodu, den je otevřen.

2

Pauza nebo Odchod mimo pracoviště (volitelné)

Čas pauzy a čas mimo pracoviště se zaznamenají a odečtou od odpracovaného času.

3

Ukončení směny → den uzavřen

Vypočítá se odpracovaný čas. Nelze ukončit v pauze nebo mimo pracoviště.

9.3 Nastavení hesla přes e-mail
1

Admin/vedoucí vytvoří uživatele

Backend uloží uživatele a vygeneruje SHA-256 hash jednorázového tokenu.

2

E-mail s odkazem odeslán zaměstnanci

PHPMailer odešle odkaz s tokenem. Token samotný v DB není — uložen je jen jeho hash.

3

Zaměstnanec nastaví heslo

Token se ověří, heslo uloží jako hash, token je označen jako použitý.

10. Bezpečnostní návrh

🔑

Hesla

Ukládána pomocí password_hash (bcrypt). Ověřování přes password_verify. Nikdy v čitelné podobě.

🍪

Session autentizace

Po přihlášení uložena identita do PHP session. Backend ověřuje session u každého chráněného endpointu.

🛡️

Autorizace dle role

Funkce v auth_helpers.php kontrolují roli a příslušnost k firmě. Vedoucí nemůže přistupovat k cizím datům.

🎫

Tokeny hesel

V DB uložen pouze SHA-256 hash. Token je jednorázový, časově omezený a po použití zneplatněn.

✅

Validace vstupů

Validace probíhá na frontendu i backendu — povinná pole, pořadí dat, rozsah dovolené, existující entita.

📁

Ukládání souborů

Fotky zaměstnanců ukládány do backend/uploads/users/. V DB uložena jen relativní cesta.

14. Nasazení a spuštění

Backend

  • Zkopírovat obsah backend/ do C:\wamp64\www\api\
  • Spustit WAMP, vytvořit databázi chronisdb, importovat SQL nebo spustit migrační skripty ze backend/sql/
  • Zkontrolovat nastavení SMTP v backend/config.env

Frontend

  • Spustit vývojový server: cd frontend && npm install && npm run dev
  • Aplikace dostupná na http://localhost:5173
  • Build pro produkci: npm run build → výstup v frontend/dist/

16 & 17. Rozšiřitelnost a omezení

Systém je připraven na budoucí rozšíření: detailní plánování směn, auditní log, automatické notifikace, napojení na čtečku čipů, produkční HTTPS a automatické zálohy databáze.

Aktuální omezení: backend bez frameworku, optimalizace pro lokální provoz a školní prezentaci, produkční bezpečnost by vyžadovala izolaci secrets mimo repozitář.
